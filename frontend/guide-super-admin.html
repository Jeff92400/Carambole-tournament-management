Guide Super Admin — Création et Onboarding d'un CDB

Procédure complète pour créer un nouveau Comité Départemental de Billard sur la plateforme, de l'import des données FFB à la mise en service.

← Retour au tableau de bord Super Admin

1. Introduction

Ce guide décrit la procédure complète pour créer et configurer un nouveau CDB (Comité Départemental de Billard) sur la plateforme de gestion des tournois. Il est destiné exclusivement au Super Admin de la plateforme.

Objectif

À l'issue de cette procédure, le nouveau CDB disposera de :

  • Une organisation active avec ses clubs et joueurs importés depuis les données FFB
  • Un compte administrateur pour le responsable du CDB
  • Des adresses email de communication configurées automatiquement
  • Un accès à l'Espace Joueur (Player App) pour les licenciés

Pré-requis

  • Être connecté en tant que Super Admin
  • Disposer des fichiers FFB à jour (Ligues, Clubs, Licences)
  • Connaître le code CDB de l'organisation à créer (ex : 0692, 9394)
  • Avoir identifié le futur administrateur du CDB (nom, licence FFB)

2. Pré-requis : Données FFB

Avant de créer un CDB, il est impératif d'avoir importé les fichiers de données FFB sur la plateforme. Ces données servent à :

  • Peupler la liste des CDB disponibles à la création
  • Identifier les clubs rattachés au CDB
  • Importer automatiquement les joueurs licenciés dans le nouveau CDB

Import des fichiers FFB

  1. Accéder à Super Admin > FFB Fichiers
  2. Importer les trois fichiers CSV dans l'ordre suivant :
    OrdreFichierContenu
    1LiguesListe des ligues régionales
    2ClubsClubs rattachés à chaque ligue et CDB
    3LicencesJoueurs licenciés avec clubs, classements et moyennes
  3. Vérifier les compteurs sur le Tableau de bord Super Admin (nombre de ligues, CDB, clubs, licences)
Important : L'ordre d'import est crucial. Les licences référencent les clubs, qui eux-mêmes référencent les ligues. Importer dans le désordre peut entraîner des données incohérentes.

Vérifier le domaine email

Les adresses email du nouveau CDB sont générées automatiquement à partir du domaine email de la plateforme.

  1. Accéder à Super Admin > Paramètres
  2. Vérifier que le champ Domaine email de la plateforme est renseigné (ex : carambole-gestion.fr)
  3. S'assurer que ce domaine est vérifié dans Resend (enregistrements DNS DKIM + SPF configurés)
Attention : Si le domaine n'est pas vérifié dans Resend, les emails envoyés par le nouveau CDB seront silencieusement rejetés. L'API Resend retourne un statut 200 (succès) même pour un domaine non vérifié — les emails ne sont simplement jamais délivrés.

3. Créer un nouveau CDB — Étape par étape

Étape 1 : Sélection du CDB

  1. Accéder à Super Admin > CDBs
  2. Cliquer sur le bouton « Créer un CDB »
  3. Dans le formulaire de création :
    • Sélectionner la Ligue — Choisir la ligue régionale de rattachement
    • Sélectionner le(s) code(s) CDB — La liste affiche uniquement les CDB de la ligue sélectionnée qui ne sont pas encore créés sur la plateforme. Il est possible de sélectionner plusieurs codes CDB pour regrouper des comités (ex : 93 + 94 = CDB 9394).

Étape 2 : Informations de l'organisation

Après sélection du CDB, les champs suivants sont pré-remplis automatiquement à partir des données FFB :

ChampDescriptionExemple
Nom completNom officiel de l'organisationComité Départemental de Billard des Hauts-de-Seine
Nom courtAbréviation pour l'affichageCDBHS
SlugIdentifiant technique (URL, emails, fichiers)cdbhs
Le slug est permanent ! Une fois le CDB créé, le slug ne peut plus être modifié. Il est utilisé dans les URLs, les adresses email et les chemins de fichiers. Vérifiez-le soigneusement avant de valider.

Étape 3 : Administrateur du CDB

  1. Rechercher le licencié FFB qui sera administrateur du CDB — saisir son nom ou numéro de licence dans le champ de recherche
  2. Sélectionner le joueur dans les résultats
  3. Définir les identifiants du compte administrateur :
    • Nom d'utilisateur — pré-rempli, modifiable
    • Mot de passe — définir un mot de passe temporaire
Sécurité : Communiquez le mot de passe temporaire par SMS ou appel téléphonique, jamais par email. L'administrateur devra le changer à sa première connexion.

Étape 4 : Résumé et finalisation

  1. Cliquer sur « Créer le CDB »
  2. Un résumé s'affiche avec les informations créées :
    • Nombre de clubs importés depuis les données FFB
    • Nombre de joueurs importés (licenciés rattachés aux clubs du CDB)
    • URL de connexion du CDB (pour l'administrateur)
    • URL de l'Espace Joueur (Player App)
  3. Copier les URLs affichées — elles seront nécessaires pour communiquer à l'administrateur
  4. Optionnel : cliquer sur « Envoyer le mail de bienvenue » pour notifier l'administrateur avec ses identifiants et les URLs
Création terminée ! Le CDB est maintenant actif sur la plateforme. L'administrateur peut se connecter et commencer à configurer son environnement.

4. Points d'attention critiques

PointDétailImpact
Slug permanent Le slug est utilisé dans les URLs, les adresses email (slug@domaine) et les chemins de fichiers (logos des clubs). Il ne peut pas être modifié après création. Irréversible
Adresses email auto-générées Trois adresses sont créées automatiquement : {slug}@{domaine} pour la communication, les convocations et le no-reply. Elles sont modifiables ultérieurement dans les paramètres du CDB. Configurable
Domaine Resend vérifié Le domaine email doit être vérifié dans Resend (DNS DKIM + SPF). Sans vérification, les emails sont silencieusement rejetés (pas d'erreur visible). Bloquant
Mot de passe admin Le mot de passe initial doit être communiqué par un canal sécurisé (SMS, téléphone). Ne jamais l'envoyer par email. Sécurité
Qualité des données FFB Les clubs et joueurs importés dépendent directement de la qualité du fichier FFB Licences. Des données manquantes ou incorrectes (ex : clubs sans code CDB) entraîneront des joueurs ou clubs non importés. Qualité
URL Espace Joueur L'URL de la Player App est automatiquement enregistrée dans les paramètres du CDB et disponible comme variable {player_app_url} dans le modèle de mail de bienvenue. Automatique

5. Post-création : Checklist administrateur CDB

Une fois le CDB créé, l'administrateur du CDB doit effectuer les actions suivantes pour finaliser la configuration. Transmettez-lui cette liste.

Actions prioritaires

  • Changer le mot de passe — Paramètres > Mon profil > Modifier le mot de passe
  • Configurer le branding — Paramètres > Apparence : couleurs, logo de l'organisation
  • Vérifier les adresses email — Paramètres > Emails : adresses de communication, convocations, no-reply
  • Configurer les paramètres de jeu — Paramètres > Paramètres de jeu : distances et reprises par catégorie

Configuration complémentaire

  • Créer les utilisateurs — Utilisateurs : ajouter les éditeurs et consultants si nécessaire
  • Vérifier les clubs importés — Clubs : vérifier la liste, ajouter les logos des clubs
  • Vérifier les joueurs importés — Joueurs : vérifier les licenciés, compléter les informations manquantes (emails, téléphones)
  • Créer le calendrier — Calendrier : importer ou créer manuellement les tournois de la saison
  • Configurer les catégories — Paramètres > Catégories : activer/désactiver les catégories utilisées
  • Tester l'envoi d'email — Envoyer un email de test depuis Emailing pour vérifier que le domaine Resend fonctionne

6. Synchronisation FFB

La synchronisation FFB permet de mettre à jour les joueurs d'un CDB existant avec les dernières données du fichier FFB Licences.

Quand synchroniser ?

  • En début de saison — pour intégrer les nouveaux licenciés et mettre à jour les classements
  • Après un nouvel import du fichier Licences — pour refléter les modifications FFB
  • Quand un CDB signale des joueurs manquants dans sa liste

Procédure

  1. S'assurer que le dernier fichier FFB Licences a été importé (Super Admin > FFB Fichiers)
  2. Accéder à Super Admin > CDBs
  3. Sur la ligne du CDB concerné, cliquer sur « Sync FFB »
  4. Un aperçu s'affiche montrant les différences détectées :
    • Nouveau Joueurs à ajouter (nouveaux licenciés)
    • Modifié Joueurs avec des informations mises à jour (classement, club, etc.)
  5. Vérifier les changements proposés
  6. Cliquer sur « Appliquer la synchronisation » pour confirmer
Note : La synchronisation n'efface jamais de joueurs existants. Elle ajoute les nouveaux et met à jour les informations des joueurs existants (classement, club de rattachement, etc.).

7. Suppression d'un CDB

Action irréversible ! La suppression d'un CDB entraîne la suppression définitive de toutes ses données. Cette opération ne peut pas être annulée.

Données supprimées

La suppression d'un CDB entraîne la suppression de :

  • L'organisation et ses paramètres
  • Tous les joueurs rattachés au CDB
  • Tous les clubs et leurs logos
  • Tous les tournois (calendrier et résultats)
  • Toutes les inscriptions
  • Tous les classements saisonniers
  • Tous les utilisateurs (comptes admin, éditeur, consultants)
  • Tous les comptes joueurs (Player App)
  • Toutes les campagnes email et modèles
  • Toutes les catégories et paramètres de jeu

Procédure

  1. Accéder à Super Admin > CDBs
  2. Cliquer sur le bouton de suppression sur la ligne du CDB
  3. Confirmer en saisissant le nom du CDB (sécurité anti-erreur)
  4. La suppression est exécutée immédiatement

8. Dépannage

ProblèmeCause probableSolution
Aucun CDB disponible dans la liste de création Fichier FFB Clubs non importé, ou tous les CDB de la ligue sont déjà créés Vérifier l'import FFB dans FFB Fichiers. Sélectionner une autre ligue.
0 joueur importé après la création Fichier FFB Licences non importé, ou les clubs du CDB n'ont pas de licenciés dans le fichier Importer le fichier Licences via FFB Fichiers, puis utiliser Sync FFB sur le CDB.
Le licencié recherché n'apparaît pas (recherche admin) Le joueur n'est pas dans le fichier FFB Licences, ou la recherche ne correspond pas Vérifier le fichier Licences. Essayer avec le numéro de licence exact.
Les emails ne sont pas reçus Domaine email non vérifié dans Resend Vérifier la configuration DNS (DKIM + SPF) du domaine dans Resend > Domains. L'API retourne 200 même si le domaine n'est pas vérifié.
L'administrateur ne peut pas se connecter Mauvaise URL de connexion, ou mot de passe incorrect Vérifier l'URL : https://[app]/login.html?org=[slug]. Réinitialiser le mot de passe si nécessaire via Super Admin > Utilisateurs.
Les couleurs/branding d'un autre CDB s'affichent Cache navigateur avec les anciennes données Vider le cache du navigateur ou forcer un rafraîchissement (Ctrl+Shift+R / Cmd+Shift+R).
Slug incorrect après création Le slug a été validé avec une erreur Le slug est permanent. La seule solution est de supprimer le CDB et de le recréer avec le bon slug.
