1. Vue d'ensemble

Qu'est-ce que le Super Admin ?

Le Super Admin (SA) est le rôle de plus haut niveau de la plateforme. Il permet de :

  • Gérer l'ensemble des Comités Départementaux de Billard (CDB) sur une plateforme centralisée
  • Importer et maintenir les données officielles de la Fédération Française de Billard (FFB)
  • Créer et configurer de nouveaux CDBs
  • Gérer les utilisateurs à travers toutes les organisations
  • Accéder à n'importe quel CDB pour assistance ou vérification
💡 Important : Le Super Admin a accès à toutes les données de tous les CDBs. Ce rôle doit être réservé aux administrateurs de confiance au niveau national FFB.

Responsabilités du Super Admin

Domaine Responsabilités
Données FFB Import des fichiers CSV officiels (Ligues, Clubs, Licences)
CDBs Création, configuration, activation/désactivation
Utilisateurs Gestion des comptes admins CDB, admins ligue, droits d'accès
Support Assistance technique, réinitialisation mots de passe, dépannage
Plateforme Paramètres globaux, domaine email, templates de bienvenue

2. Architecture de la plateforme

Hiérarchie des données FFB

La plateforme reflète la structure officielle de la FFB :

FFB (Fédération) └─ Ligues (16 ligues régionales) └─ CDBs (~88 comités départementaux) └─ Clubs (~590 clubs) └─ Licenciés (~20 000 joueurs)

Multi-organisation (Multi-CDB)

Chaque CDB dispose de son propre environnement complètement isolé :

Élément Isolation Exemple
Joueurs ✅ Séparés par CDB Les joueurs du CDB 92 ne sont pas visibles par le CDB 93
Compétitions ✅ Séparées par CDB Chaque CDB gère ses propres tournois
Classements ✅ Séparés par CDB Les classements sont calculés indépendamment
Paramètres ✅ Personnalisables Logo, couleurs, emails, règles de notation
Utilisateurs ✅ Assignés à un CDB Un admin ne peut gérer que son CDB
✓ Avantage : Cette isolation garantit que chaque CDB travaille de manière autonome, sans risque de contamination de données entre départements.

Données partagées

Certaines données sont partagées au niveau national :

  • Ligues FFB - Liste des 16 ligues régionales
  • CDBs FFB - Liste des comités départementaux
  • Clubs FFB - Référentiel national des clubs
  • Licenciés FFB - Base nationale des licences actives
💡 Note : Ces données partagées sont en lecture seule pour les CDBs. Seul le Super Admin peut les importer et les mettre à jour.

3. Gestion des CDBs

Créer un nouveau CDB

Étape 1 : Accéder à la page CDBs

Depuis le tableau de bord Super Admin, cliquez sur CDBs dans la barre de navigation.

Étape 2 : Cliquer sur "Créer un CDB"

Un formulaire s'ouvre avec les champs suivants :

  • CDB FFB - Sélectionner dans la liste des CDBs officiels (ex: "92 - HAUTS DE SEINE")
  • Nom complet - Auto-rempli depuis FFB (ex: "Comité Départemental de Billard des Hauts-de-Seine")
  • Nom court - Version abrégée (ex: "CDBHS")
  • Slug - Identifiant URL unique (ex: "cdbhs")
  • Admin - Rechercher un licencié - Chercher le futur admin dans les licences FFB
  • Nom d'utilisateur admin - Login pour le premier admin
  • Email admin - Email du premier admin
  • Mot de passe admin - Mot de passe initial

Étape 3 : Validation et création

Lors de la création, la plateforme effectue automatiquement :

  1. Création de l'organisation dans la base
  2. Génération des adresses email (ex: cdbhs@carambole-gestion.fr)
  3. Création du compte admin initial
  4. Copie des catégories de référence depuis le CDB #1
  5. Initialisation des paramètres par défaut
⚠️ Important : Le slug choisi sera utilisé dans l'URL Player App (?org=slug). Il ne peut pas être modifié après création.

Peupler les joueurs du CDB (Seeding)

Après création d'un CDB, il faut importer les joueurs depuis la base FFB :

Option 1 : Import depuis la page CDBs

  1. Sur la ligne du CDB, cliquez sur "Importer joueurs FFB"
  2. Une prévisualisation affiche les joueurs FFB du département (ex: 150 licenciés)
  3. Validez pour importer tous les joueurs dans la table players du CDB

Option 2 : Import manuel par l'admin CDB

L'admin du CDB peut également importer ses joueurs via Paramètres → Import joueurs en uploadant un fichier CSV.

Synchroniser les joueurs FFB

Au fil de la saison, de nouveaux licenciés arrivent ou des données changent (classements, clubs). Pour synchroniser :

  1. Cliquez sur "Sync FFB" sur la ligne du CDB
  2. La prévisualisation affiche :
    • ✓ Joueurs à jour
    • ⚠ Joueurs à mettre à jour (classement ou club modifié)
    • + Nouveaux licenciés (pas encore dans le CDB)
  3. Validez pour appliquer les changements
💡 Conseil : Effectuez une synchronisation en début de saison (septembre) et une à mi-saison (janvier) pour garder les données à jour.

Envoyer l'email de bienvenue

Après configuration du CDB, envoyez l'email de bienvenue à l'admin :

  1. Sur la ligne du CDB, cliquez sur l'icône email (✉️)
  2. Le template de bienvenue pré-rempli s'affiche (configurable dans Paramètres)
  3. Vérifiez le contenu et envoyez
  4. L'admin recevra :
    • Lien vers le dashboard du CDB
    • Lien vers le Player App
    • Identifiants de connexion
    • Guide d'utilisation

Désactiver / Réactiver un CDB

Pour désactiver temporairement un CDB (ex: non-paiement, CDB dissout) :

  1. Cliquez sur le bouton toggle "Actif / Inactif"
  2. Un CDB inactif :
    • ❌ Les admins ne peuvent plus se connecter
    • ❌ Le Player App affiche un message "CDB non disponible"
    • ✅ Les données sont préservées (pas de suppression)
  3. Réactivation possible à tout moment

Supprimer un CDB

⚠️ DANGER - Action irréversible !
La suppression d'un CDB efface TOUTES ses données :
  • Joueurs, compétitions, résultats, classements
  • Utilisateurs, emails, paramètres
  • Logs d'activité

Il n'y a aucun moyen de récupérer ces données après suppression.

Processus de suppression :

  1. Cliquez sur le bouton "Supprimer" (icône poubelle)
  2. Une confirmation vous demande de taper le nom du CDB
  3. La suppression est immédiate et définitive
💡 Alternative : Pour conserver les données historiques, préférez désactiver le CDB plutôt que le supprimer.

4. Données FFB

Sources de données officielles

La FFB fournit des fichiers CSV officiels contenant :

Fichier Contenu Fréquence de mise à jour
Ligues.csv 16 ligues régionales Rarement (structure stable)
Clubs.csv ~590 clubs nationaux Annuelle (nouveaux clubs, fermetures)
Licences.csv ~20 000 licenciés actifs Mensuelle ou trimestrielle

Importer les fichiers FFB

Depuis FFB Fichiers dans la barre de navigation :

Import Ligues

  1. Cliquez sur "Choisir un fichier" sous la section Ligues
  2. Sélectionnez Ligues.csv
  3. Cliquez sur "Importer"
  4. Résultat : 16 ligues importées ou mises à jour

Import Clubs

  1. Sélectionnez Clubs.csv
  2. Le système extrait automatiquement le numéro de ligue et le code CDB
  3. Résultat : ~590 clubs importés, liés à leurs ligues et CDBs

Import Licences

  1. Sélectionnez Licences.csv
  2. Pour chaque licence, le système enregistre :
    • Numéro de licence
    • Nom, prénom, email
    • Ligue, CDB, club
    • Classements (Libre, Cadre, Bande, 3 Bandes)
  3. Résultat : ~20 000 licenciés importés
💡 Ordre d'import : Importez toujours dans cet ordre : Ligues → Clubs → Licences. Les licences dépendent des clubs, qui dépendent des ligues.

Consulter les données FFB

Page Données FFB pour explorer les données importées :

  • Onglet Ligues - Liste des 16 ligues avec nombre de clubs et licenciés
  • Onglet CDBs - Liste des ~88 CDBs avec leur ligue d'appartenance
  • Onglet Clubs - Liste des ~590 clubs avec filtres par ligue/CDB
  • Onglet Licenciés - Recherche de licenciés avec filtres avancés

Utilisation des données FFB

Les données FFB servent de référentiel pour :

  • Création de CDBs - Sélection dans la liste officielle des CDBs
  • Recherche d'admins - Recherche de licenciés FFB pour créer un compte admin
  • Seeding joueurs - Import des licenciés d'un département dans un CDB
  • Synchronisation - Mise à jour des classements et changements de clubs
⚠️ Protection des données : Les données FFB sont confidentielles. L'accès est réservé au Super Admin. Les CDBs ne voient que leurs propres joueurs.

5. Gestion des Ligues

Vue d'ensemble

La page Ligues permet de gérer les 16 ligues régionales FFB.

Informations de contact

Pour chaque ligue, vous pouvez ajouter :

  • Email de contact
  • Téléphone
  • Site web
  • Adresse postale
  • Logo de la ligue

Modifier les informations d'une ligue

  1. Cliquez sur le bouton "Modifier" (icône crayon) sur la ligne de la ligue
  2. Remplissez les champs souhaités
  3. Pour le logo :
    • Formats acceptés : PNG, JPG, SVG
    • Taille recommandée : 200x200 pixels
    • Poids max : 2 MB
  4. Cliquez sur "Enregistrer"

Admins de ligue (à venir)

💡 Fonctionnalité future : Les admins de ligue auront accès à un tableau de bord regroupant les statistiques et calendriers de tous les CDBs de leur ligue.

Actuellement, vous pouvez créer un compte "Admin Ligue" via Utilisateurs → Créer Admin Ligue.

6. Gestion des utilisateurs

Types d'utilisateurs

Rôle Badge Accès Utilisation
Super Admin SA Plateforme entière, tous CDBs Administrateur national FFB
Admin Admin Un seul CDB, accès complet Administrateur d'un CDB
Lecteur Lecteur Un seul CDB, lecture seule Audit, consultation sans modification
Viewer Viewer Un seul CDB, lecture limitée Consultation basique (tournois, classements)
Admin Ligue Ligue Tous CDBs de sa ligue Responsable régional (future fonctionnalité)

Voir tous les utilisateurs

Page Utilisateurs - Vue d'ensemble :

  • KPIs en haut :
    • Nombre total d'utilisateurs
    • Nombre de Super Admins
    • Répartition par rôle
    • Répartition par CDB
  • Filtres :
    • Recherche par nom d'utilisateur ou email
    • Filtre par CDB
    • Filtre par rôle
    • Filtre par statut (Actif / Inactif)
  • Tableau : Liste de tous les utilisateurs de tous les CDBs

Créer un utilisateur pour un CDB

Depuis la page Utilisateurs

  1. Cliquez sur "Créer un utilisateur"
  2. Remplissez :
    • Nom d'utilisateur (unique)
    • Email (optionnel mais recommandé)
    • Mot de passe (minimum 6 caractères)
    • Rôle (Admin, Viewer, Lecteur)
    • CDB (sélectionner le CDB d'appartenance)
  3. Cliquez sur "Créer"
💡 Mot de passe initial : Communiquez le mot de passe à l'utilisateur de manière sécurisée (téléphone, email chiffré). Recommandez-lui de le changer immédiatement après première connexion.

Modifier un utilisateur

Cliquez sur une ligne du tableau ou sur le bouton "Modifier" :

  • Nom d'utilisateur - Non modifiable (en lecture seule)
  • Email - Modifiable
  • Rôle - Modifiable (Admin, Viewer, Lecteur)
  • CDB - Réassignable (changer l'utilisateur de CDB)
  • Super Admin - Case à cocher pour donner/retirer les droits SA
  • Compte actif - Activer/désactiver le compte
  • Mot de passe - Section dédiée (voir ci-dessous)

Réinitialiser un mot de passe

Dans le formulaire de modification d'utilisateur :

Section Mot de passe

Affiche :

  • Statut : ✓ Défini (tous les utilisateurs existants ont un mot de passe)
  • Dernière modification : Date de la dernière réinitialisation, ou "Jamais"

Pour réinitialiser :

  1. Cochez "Réinitialiser le mot de passe"
  2. Deux champs apparaissent :
    • Nouveau mot de passe
    • Confirmer mot de passe
  3. Saisissez le nouveau mot de passe (minimum 6 caractères)
  4. Confirmez-le en le retapant
  5. Cliquez sur "Enregistrer"
💡 Quand utiliser cette fonction ?
  • Un utilisateur a perdu/oublié son mot de passe
  • L'email de l'utilisateur est invalide (fonction "Mot de passe oublié" ne marche pas)
  • Urgence : besoin d'accès immédiat

Méthode normale : L'utilisateur utilise "Mot de passe oublié ?" sur la page de connexion (réception d'un code par email).

Activer / Désactiver un utilisateur

Deux méthodes :

  • Depuis le tableau : Cliquez sur le bouton 🚫 (Désactiver) ou ✅ (Activer)
  • Depuis le formulaire : Décochez/Cochez "Compte actif" et enregistrez

Effet d'un compte inactif :

  • ❌ Connexion impossible (message "Compte désactivé")
  • ✅ Données préservées (logs d'activité conservés)
  • ✅ Réactivation possible à tout moment

Donner/Retirer les droits Super Admin

⚠️ Attention : Le rôle Super Admin donne accès à TOUTES les données de TOUS les CDBs. À réserver aux administrateurs de confiance.

Pour donner les droits SA :

  1. Ouvrez l'utilisateur en modification
  2. Cochez "Super Admin (accès complet à toutes les organisations)"
  3. Enregistrez
  4. L'utilisateur a maintenant accès au menu Super Admin

Un Super Admin peut également avoir un CDB d'appartenance (optionnel). Cela définit son contexte par défaut à la connexion.

7. Paramètres plateforme

Domaine email de la plateforme

Page Paramètres - Section "Domaine email plateforme" :

Configurez le domaine utilisé pour générer les adresses email des nouveaux CDBs.

Configuration actuelle (exemple)

  • Domaine : carambole-gestion.fr
  • Effet : Lors de la création du CDB "CDBHS" (slug: cdbhs), les adresses suivantes sont auto-générées :
    • cdbhs@carambole-gestion.fr (communication)
    • convocations-cdbhs@carambole-gestion.fr (convocations)
    • noreply-cdbhs@carambole-gestion.fr (emails automatiques)
⚠️ Configuration DNS requise :
  • Le domaine doit être vérifié dans Resend (plateforme d'envoi d'emails)
  • Les enregistrements DKIM et SPF doivent être configurés
  • Voir documentation Resend pour les détails techniques

Email de bienvenue CDB

Personnalisez l'email envoyé aux admins des nouveaux CDBs :

Éditeur de template

Utilisez l'éditeur riche (Quill) pour composer l'email avec :

  • Mise en forme (gras, italique, listes)
  • Variables dynamiques (voir tableau ci-dessous)
  • Instructions de connexion
  • Liens vers les guides

Variables disponibles

Variable Description Exemple
{organization_name} Nom complet du CDB Comité Départemental de Billard des Hauts-de-Seine
{organization_short_name} Nom court CDBHS
{admin_username} Login de l'admin admin92
{admin_email} Email de l'admin contact@cdbhs.net
{dashboard_url} Lien vers le dashboard https://app.com/login.html?org=cdbhs
{player_app_url} Lien vers le Player App https://player.app/?org=cdbhs

Envoyer un email de test

  1. Composez votre template avec les variables
  2. Cliquez sur "Envoyer un test"
  3. Entrez votre email
  4. Vous recevrez l'email avec les variables remplacées par des valeurs d'exemple
✓ Conseil : Incluez dans l'email de bienvenue :
  • Identifiants de connexion
  • Lien vers le guide utilisateur
  • Lien vers le Player App (à partager avec les joueurs)
  • Contacts support
  • Prochaines étapes recommandées (import joueurs, paramétrage)

8. Navigation et accès

Menu Super Admin

Lorsque vous êtes connecté en tant que Super Admin, la barre de navigation affiche :

Onglet Page Description
📊 Tableau de bord KPIs plateforme, statistiques FFB, enrôlements récents
🏢 CDBs Gestion des comités départementaux
🏛️ Ligues Gestion des ligues régionales
📊 Données FFB Consultation des données FFB importées
📁 FFB Fichiers Import des CSV FFB (Ligues, Clubs, Licences)
👥 Utilisateurs Gestion de tous les utilisateurs plateforme
⚙️ Paramètres Domaine email, template de bienvenue

Dropdown CDB

En haut à droite de la barre de navigation, un dropdown CDB permet de :

  • Voir la liste de tous les CDBs actifs
  • Cliquer sur un CDB pour accéder directement à son dashboard
  • Naviguer entre les contextes SA et CDB

Accéder au dashboard d'un CDB

  1. Cliquez sur le dropdown CDB (en haut à droite)
  2. Sélectionnez le CDB souhaité (ex: "CDBHS")
  3. Vous êtes redirigé vers le dashboard du CDB
  4. Vous voyez maintenant :
    • Les joueurs de ce CDB uniquement
    • Les compétitions de ce CDB
    • Les classements de ce CDB
    • Les paramètres de ce CDB
💡 Contexte CDB vs SA :
  • Pages SA (super-admin-*.html) → Données de tous les CDBs
  • Pages CDB (dashboard.html, etc.) → Données d'un seul CDB

Le Super Admin peut naviguer librement entre ces deux contextes.

Retour au niveau SA

Depuis le dashboard d'un CDB, pour revenir au niveau Super Admin :

  • Option 1 : Cliquez sur un des onglets SA dans le menu (Tableau de bord SA, CDBs, etc.)
  • Option 2 : Utilisez le dropdown CDB et sélectionnez "Niveau Super Admin"

9. Flux de travail courants

Scénario 1 : Onboarding d'un nouveau CDB

Workflow complet

  1. Importer les données FFB (si pas déjà fait) :
    • FFB Fichiers → Importer Ligues.csv
    • FFB Fichiers → Importer Clubs.csv
    • FFB Fichiers → Importer Licences.csv
  2. Créer le CDB :
    • CDBs → Créer un CDB
    • Remplir le formulaire (sélection FFB CDB, admin, etc.)
    • Valider la création
  3. Importer les joueurs FFB :
    • Sur la ligne du CDB → Importer joueurs FFB
    • Prévisualisation → Valider
  4. Envoyer l'email de bienvenue :
    • Sur la ligne du CDB → Icône email ✉️
    • Vérifier le contenu → Envoyer
  5. Assistance initiale :
    • Accéder au dashboard du CDB (dropdown CDB)
    • Vérifier que les joueurs sont bien importés
    • Aider l'admin à paramétrer (logo, couleurs, emails)

Scénario 2 : Début de saison (septembre)

Actions recommandées

  1. Mettre à jour les données FFB :
    • Recevoir les nouveaux fichiers CSV de la FFB
    • Importer Licences.csv (nouveaux licenciés, changements de clubs)
    • Importer Clubs.csv si modifications
  2. Synchroniser les CDBs :
    • Pour chaque CDB actif → Sync FFB
    • Appliquer les mises à jour (nouveaux joueurs, classements)
  3. Vérifier les comptes admins :
    • Utilisateurs → Vérifier que tous les admins sont actifs
    • Réinitialiser les mots de passe si besoin

Scénario 3 : Support utilisateur

Un admin CDB a oublié son mot de passe

  1. Méthode normale : Diriger l'admin vers "Mot de passe oublié ?" sur la page de connexion
  2. Si l'email est invalide :
    • Utilisateurs → Trouver l'utilisateur
    • Modifier → Mettre à jour l'email
    • Enregistrer → L'admin peut maintenant utiliser "Mot de passe oublié ?"
  3. En urgence :
    • Utilisateurs → Trouver l'utilisateur
    • Modifier → Réinitialiser le mot de passe
    • Communiquer le nouveau mot de passe par téléphone
    • Recommander à l'admin de le changer après connexion

Un admin CDB ne voit pas ses joueurs

  1. Accéder au dashboard du CDB (dropdown CDB → sélectionner le CDB)
  2. Vérifier page Joueurs → Si vide :
  3. Retour niveau SA → CDBs → Importer joueurs FFB pour ce CDB
  4. Vérifier à nouveau depuis le dashboard du CDB

Un CDB veut changer son logo/couleurs

  1. Option 1 : L'admin CDB peut le faire lui-même (Paramètres → Organisation)
  2. Option 2 : Vous le faites pour lui :
    • Dropdown CDB → Sélectionner le CDB
    • Paramètres → Organisation
    • Upload logo, modifier couleurs
    • Enregistrer

10. Dépannage

Problème : Import FFB échoue

Symptômes : Message d'erreur lors de l'import d'un fichier CSV FFB

Causes possibles :

  • Format du fichier incorrect (délimiteur, encodage)
  • Colonnes manquantes ou renommées
  • Données manquantes (ex: ligue_numero vide dans Clubs.csv)

Solutions :

  1. Vérifier le format CSV : délimiteur point-virgule ;, encodage UTF-8
  2. Ouvrir le fichier dans un éditeur de texte pour vérifier les en-têtes
  3. Comparer avec les fichiers précédents (structure identique ?)
  4. Tester avec un échantillon de 10 lignes d'abord
  5. Consulter les logs serveur pour le message d'erreur exact

Problème : Un CDB ne reçoit pas les emails

Symptômes : Emails de bienvenue ou notifications non reçus

Vérifications :

  1. Vérifier que l'adresse email du destinataire est correcte (Utilisateurs → Modifier)
  2. Vérifier le dossier spam/courrier indésirable
  3. Vérifier que le domaine email est bien vérifié dans Resend :
    • Se connecter à Resend → Domains
    • Vérifier que le domaine a le statut "Verified" ✅
    • Si non vérifié, configurer les enregistrements DNS (DKIM, SPF)
  4. Tester avec une adresse email personnelle (Gmail, Outlook)
⚠️ Limitation Resend (plan gratuit) :
  • 1 seul domaine personnalisé
  • 100 emails / jour
  • 3 000 emails / mois

Si plusieurs CDBs partagent le même domaine email, les limites s'appliquent globalement.

Problème : Un admin ne peut pas se connecter

Symptômes : Message "Identifiants incorrects" ou "Compte désactivé"

Vérifications :

  1. Vérifier le statut du compte :
    • Utilisateurs → Rechercher l'utilisateur
    • Vérifier que "Compte actif" est coché
    • Si inactif → Activer et enregistrer
  2. Vérifier le CDB :
    • CDBs → Vérifier que le CDB est actif
    • Si inactif → Réactiver
  3. Réinitialiser le mot de passe :
    • Utilisateurs → Modifier → Réinitialiser le mot de passe
    • Communiquer le nouveau mot de passe à l'admin
  4. Vérifier l'URL de connexion :
    • URL correcte : /login.html?org=slug
    • Exemple : /login.html?org=cdbhs

Problème : Les données d'un CDB se mélangent avec un autre

Symptômes : Un admin voit les joueurs/tournois d'un autre CDB

Cause : Contamination de données (bug critique d'isolation)

⚠️ ALERTE SÉCURITÉ : Ce problème ne devrait jamais arriver. Si observé, contactez immédiatement le développeur.

Vérifications immédiates :

  1. Vérifier les logs d'activité du CDB concerné
  2. Vérifier l'organization_id des données contaminées
  3. Arrêter toute opération sur la plateforme
  4. Créer un backup de la base de données
  5. Analyser le code et corriger le bug avant reprise

Problème : La synchronisation FFB ne trouve pas de nouveaux joueurs

Symptômes : Sync FFB affiche "0 nouveaux licenciés" alors qu'il devrait y en avoir

Causes possibles :

  • Le fichier Licences.csv n'a pas été réimporté
  • Le code CDB dans le fichier FFB ne correspond pas au code CDB de l'organisation
  • Les joueurs ont déjà été importés (vérifier la table players du CDB)

Solutions :

  1. Vérifier la date du dernier import FFB (Données FFB → onglet Licenciés)
  2. Réimporter Licences.csv si nécessaire
  3. Vérifier le code FFB CDB dans CDBs → Modifier CDB
  4. Comparer manuellement : nombre de licenciés FFB vs nombre de joueurs dans le CDB
